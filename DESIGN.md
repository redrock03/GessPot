# DESIGN.md — המוח העיצובי

> **לקלוד-קוד:** זהו מקור-האמת היחיד לכל החלטה חזותית במוצר. קרא אותו לפני כל עבודת UI.
> אל תמציא צבע, גודל או רכיב מחוץ למה שמוגדר כאן. `theme.ts` ומשתני ה-CSS ב-`:root` נגזרים מהקובץ הזה ושומרים על שמות-הטוקנים. הרפרנס הוויזואלי המרונדר: `מערכת_עיצוב_מוקד_הניחושים.html`. החלטות-על וארכיטקטורה: `CLAUDE.md`.
>
> **וייב:** "קונסולת אנליסט לילה" — מגרש מוצף-זרקורים בלילה. נוקטורנלי, ממוקד, מדויק. עברית · RTL · mobile-first (PWA, ~390px). הבהירות חשובה מהקישוט; הנועזות מבוזבזת במקום אחד — **זוג ההצעות** — והשאר שקט.

---

## 1. עקרונות ליבה

1. **מבנה מקודד-מהות, לא קישוט.** צבע, תגי-שלב, אינדיקטור-ערך ו"דלתות" ההעפלה נושאים משמעות. אל תוסיף מספור או אייקון ללא רצף/מידע אמיתי.
2. **שני צירי-צבע נפרדים שאסור לערבב:** ציר **מהות** (מי מנצח): אמרלד / ענבר / ורד · ציר **סוג-הצעה** (כיוון/בול): ציאן / זהב.
3. **כל המספרים ב-mono** עם `tabular-nums` — תוצאות, אחוזים, EV, שעות, נקודות.
4. **פחות אנימציה = פחות תחושת "נוצר אוטומטית".** מעברים קצרים בלבד, וכיבוד `prefers-reduced-motion`.
5. **קופי מצד המשתמש:** פעלים פשוטים, בלי מילוי. שגיאה מסבירה מה קרה ומה לעשות. מסך ריק = הזמנה לפעולה.

## 4. מצאי רכיבים

מצבי-בסיס לכל אינטראקטיבי: `default / hover / active / focus-visible / disabled / loading`. פוקוס תמיד `box-shadow:0 0 0 2px var(--focus)`.

### 4.1 `SuggestionCard` — החתימה (כיוון / בול / converged)

שני כרטיסים-תאומים. הרכיב היחיד שמותר לו להיות נועז. מבנה: אייקון-זהות + שם + סב-כותרת · תוצאה ענקית (mono) · 3 שורות-דאטה (סיכוי-כיוון, סיכוי-בול, EV) עם מפריד לפני ה-EV.

| וריאנט      | זהות       | אייקון                             | מסגרת                                  | זוהר hover         |
| ----------- | ---------- | ---------------------------------- | -------------------------------------- | ------------------ |
| `kivun`     | `--steady` | ריבוע + טבעת ציאן + ליבה           | `--bd-steady`                          | `--sh-glow-steady` |
| `bul`       | `--strike` | **בולסאיי** (שתי טבעות זהב + ליבה) | `--bd-strike`                          | `--sh-glow-strike` |
| `converged` | ציאן→זהב   | סרט "התכנסות · ביטחון גבוה"        | מסגרת-גרדיאנט `135deg,#4DA8FF→#F5C84B` | —                  |

`hover:translateY(-2px)`. `converged` הוא מצב-נתונים (לא hover): התאומים מתאחדים לכרטיס יחיד עם סיכוי+EV משוקלל ותוצאה 52px.

> **הבחנה זהב↔ענבר:** דרך בהירות+רוויה _ובעיקר אייקונוגרפיה_ (בולסאיי מול בר-שטוח). לעולם לא דרך גוון בלבד.

### 4.2 `ProbBar` — 1X2

בר אופקי 32px, שלושה מקטעים `gap:2px`, `radius:9px`, `overflow:hidden`. רוחב לפי אחוז, רקע גרדיאנט-מהות, אחוז במרכז ב-mono כהה. מתחת: שלוש תוויות בצבעי-מהות.

### 4.3 `ScorelineCoupon` — מפת-החום (הסיכון העיצובי)

גריד `6×6` (0–5 × 0–5), `gap:5px`. **עוצמה:** `alpha=0.07+0.86*(p/pMax)`. **צבע:** `h>a→--home`, `h===a→--draw`, `h<a→--away`. טקסט תוצאה זעיר ב-mono. **סימון כיוון:** טבעת `--steady` על התא; **סימון בול:** בולסאיי `--strike`. **נבחר:** `box-shadow:0 0 0 2px #EAF1FB` + readout. צירים בצבעי-מהות (אנכי בית / אופקי חוץ).

### 4.4 `FixtureRow` — שורת-משחק

`<button>` ב-flex (RTL), `--bd`, `radius:14px`. סדר מימין: בלוק-שעה (mono)+`StageBadge` · קו-מפריד · שתי נבחרות (טוקן-קוד+שם) · הצצת-כיוון/בול (שתי תוצאות זעירות; כיוון=טבעת `--steady`, בול=עיגול-מלא `--strike`) · `ValueTag`.

- **ערך-גבוה:** פס-מבטא זהב בקצה-המוביל (`inset-inline-end:0`) + זוהר + `ValueTag` זהב + מסגרת מחוממת.
- **נעול** (`locked`): `opacity:.6`, ההצצה מוחלפת ב-`MyPickBadge` (§4.9).
- `hover` הרמה קלה; `active:scale(.99)`; `loading` שלד-שורה.

### 4.5 `StageBadge` · `ValueTag`

- **`StageBadge`:** `--surface-2`, `--bd`, `--r-xs`, 8.5–9px `--text-mute` (למשל "בתים").
- **`ValueTag`:** מרובע 26px לפי דרגת-נקודות — `pts≥3` זהב (`--strike`, רקע `rgba(245,200,75,.13)`); `pts===2` ענבר; `pts≤1` `--text-mute`/`--surface-2`. מספר ב-mono.

### 4.6 `QualDoorRow` — דלת-העפלה

אייקון-מצב + כותרת + הסבר + תג. רקע מקבל גוון-מהות עדין (`.06–.14`) ומסגרת תואמת.

| מצב          | צבע      | אייקון      | תג         | דוגמה              |
| ------------ | -------- | ----------- | ---------- | ------------------ |
| `guaranteed` | `--home` | ✓           | "מובטח"    | "תיקו מבטיח עלייה" |
| `possible`   | `--draw` | `+2` (mono) | "נצח ב-2+" | מקרה-מרווח         |
| `eliminated` | `--away` | ✕           | "מודחת"    | "הפסד/תיקו ויוצאת" |

### 4.7 `AbsenceChip` — צ'יפ-היעדרות

`flex`, `gap:8px`, `--r-lg`. שם-שחקן מודגש בצבע-המצב. תת-סוגים: **פציעה** (`--away`, רקע `#1A0E11`/מסגרת `#3a1c22`); **השעיה** (כרטיס אדום); **במרחק-כרטיס** (מלבן-כרטיס `--draw`, רקע `#1A1605`/מסגרת `#4d3f12`).

### 4.8 `TabBar` — ניווט תחתון

`flex space-around`, `border-top:1px solid #18212F`, `background:rgba(11,16,26,.92)`+`backdrop-filter:blur(12px)`. פריט: אייקון 22px + תווית 10px/700. **פעיל=`--steady`**, כבוי=`--text-faint`. גובה ≥56px, פריט ≥`--tap-min`.

### 4.9 `MyPickBadge` + מצב "הניחוש שלי" _(נוסף — תומך בנעילה-בשריקה)_

תג `--steady`: `border:1px solid #2b557d`, `background:rgba(77,168,255,.1)`, `--r-pill`, טקסט mono ("הניחוש שלי · 1-2"). מופיע ב-`FixtureRow.locked` ובראש מסך `Match` אחרי נעילה — מציג את בחירת המשתמש לצד ההצעה, ומסמן שהמשחק נעול (אחרי שריקת הפתיחה).

### 4.10 מצבי-מערכת

**Empty/Error:** אייקון במסגרת-משטח + כותרת (מה קרה) + משפט-הכוונה (מה לעשות) + כפתור ראשי (`--steady`, טקסט `--ink`). מזמין, לא מתנצל. **Loading:** שלדים בצורת-הרכיב בפעימת-`opacity` (כבוי תחת reduced-motion), לא ספינר גנרי.

## 5. נגישות (רצפה, לא תוספת)

- **ניגודיות:** `--text` על `--ink`/`--surface` ≥7:1. `--text-dim` לטקסט-משני בלבד (≥4.5:1). **אל** תשתמש ב-`--text-mute` לטקסט קריא חיוני.
- **אל תקודד מידע בצבע בלבד:** כיוון/בול נבדלים גם באייקון ובתווית; מצבי-העפלה גם בתג-טקסט.
- **פוקוס-מקלדת נראה** על כל אינטראקטיבי.
- **יעדי-מגע ≥44px.** תאי-קופון קטנים — ספק `aria-label` מלא ("מרוקו 1 קנדה 0, הסתברות 14%") ו-readout טקסטואלי.
- **`prefers-reduced-motion`** מבטל אנימציות.
- **כל מספר ב-mono+tabular-nums** — יישור-ספרות יציב.

## 6. RTL

- `dir="rtl"` על שורש-המסך; לייאאוט Flex/Grid מתהפך אוטומטית.
- השתמש בתכונות **לוגיות** (`inset-inline-start`, `margin-inline`, `text-align:start`), **למעט** מבטא-קישוט מכוון (פס ערך-גבוה: `inset-inline-end:0`).
- מספרים נשארים LTR בתוך זרימת-RTL (תקין ל-mono).
- אייקונים כיווניים (chevron) מראים לכיוון-המעבר ב-RTL.

## 8. נספח — מודל מפת-החום

```js
const pois = (l, k) => {
  let e = Math.exp(-l),
    f = 1;
  for (let i = 2; i <= k; i++) f *= i;
  return (e * Math.pow(l, k)) / f;
};
let pMax = 0;
const cells = [];
for (let h = 0; h <= 5; h++)
  for (let a = 0; a <= 5; a++) {
    const p = pois(LH, h) * pois(LA, a);
    pMax = Math.max(pMax, p);
    cells.push({ h, a, p });
  }
// תא → alpha=0.07+0.86*(p/pMax) ; color = h>a?home : h===a?draw : away
```

## 9. החלטות פתוחות (להכריע לפני נעילת הטוקנים)

1. **זהב-בול מול ענבר-תיקו** — שקול לדחוף את `--strike` מעט חם יותר (כתום-זהב) או להנמיך רוויית `--draw`, מעבר להבחנה באייקון. רלוונטי במיוחד לקופון.
2. **מיקרו-טקסט ≤10px** — שמור לקישוט; כל מספר חיוני (כולל ספרות הקופון) ל-11px+.
3. **`--text-mute` בקופון** — נכשל ב-4.5:1; שקול `--text-dim` לספרות תאים חלשים.
4. **מצב "הניחוש שלי" (§4.9)** — אומץ כתוספת. אם הפול אינו נועל בשריקה — להתאים.
